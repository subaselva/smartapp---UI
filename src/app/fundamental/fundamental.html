Bindings

In Angular, bindings are a fundamental concept that enables dynamic communication between your application's components and the view. Bindings allow you to effortlessly synchronize data and update the user interface in response to changes.

  1. Interpolation: Use double curly braces to embed component data directly into the HTML template. For example, double curly brace username will display the value of the username property.
  2. Property Binding: Bind an HTML element's property to a component property using square brackets [ ]. For instance, [src]="imageUrl" binds the imageUrl property to the src attribute of an tag.
  3. Event Binding: Respond to user interactions by binding a component method to an HTML event using parentheses ( ). For instance, (click)="handleClick()" triggers the handleClick() method when the element is clicked.
  4. Two-Way Binding: Achieve synchronization between user input and component data using the [(ngModel)] directive. It combines property binding and event binding to enable data flow in both directions.
Structural directives

*ngIf and *ngFor are called structural directives in Angular. Structural directives are used to modify the structure of the DOM by adding or removing elements based on certain conditions. They play a crucial role in dynamically rendering content in Angular templates.

  1. *ngIf Directive: This directive is used to conditionally render an element in the DOM based on a provided expression. If the expression evaluates to true, the element is included in the DOM; if it evaluates to false, the element is removed from the DOM.
  2. *ngFor Directive: This directive is used to repeat a portion of the template for each item in an iterable collection. It generates multiple instances of the element and binds them to the values from the collection.
